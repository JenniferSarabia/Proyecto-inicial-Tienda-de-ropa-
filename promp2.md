## PROMPT URBA & FLOW
Actúa como arquitecto de software senior especializado en Flutter multiplataforma y Firebase; necesito que diseñes el siguiente paso de desarrollo para la aplicación "Urba & Flow", una tienda digital de ropa para hombre con interfaz moderna en tonalidades azules, enfocada exclusivamente en entornos de desarrollo/staging (sin configuración de producción, sin analíticas, sin Crashlytics, sin despliegue a tiendas); el proyecto debe usar Dart/Flutter compatible con Android, iOS, Web y Windows, gestionar estado con Provider, enrutamiento declarativo con go_router, autenticación vía Firebase Auth (email/contraseña) con claims personalizados para dos roles (Administrador con CRUD completo de productos, variantes, inventario, pedidos, cupones, sucursales y proveedores; y Usuario/Cliente con navegación pública, carrito, checkout, historial y gestión de direcciones), y persistencia en Firestore adaptando el esquema relacional proporcionado (productos, variantes, categorías, imágenes, inventario, movimientos, clientes, direcciones, pedidos, detalles, pagos, envíos, cupones, sucursales, empleados, proveedores) a una estructura documental NoSQL con subcolecciones y campos embebidos donde corresponda, priorizando consultas eficientes, reglas de seguridad por rol, paginación, validación estricta en UI y Firestore rules, estado UI estandarizado (idle/loading/success/error), componentes responsivos/adaptativos, arquitectura feature-first, pruebas unitarias y de widget básicas, flujo Git con ramas feature/fix y documentación técnica; responde exclusivamente con arquitectura detallada, flujos de estado, estrategia de mapeo relacional→Firestore, alcance de Providers, estructura de carpetas exacta, reglas de seguridad conceptuales, rutas protegidas por rol, guía de UI/UX en azul y checklist de validación dev/staging, sin generar código fuente, manteniendo un enfoque procedural, escalable y listo para la fase de implementación.
